## Profile 套用流程

```mermaid
flowchart TD
    Start([使用者選擇 Apply 或選單切換]) --> BackupConfig[備份目前 config.toml]
    BackupConfig --> BackupName{備份檔名已存在?}
    BackupName -->|是| AddSuffix[加上遞增尾碼]
    BackupName -->|否| WriteConfig[寫入 profile TOML 到 config.toml]
    AddSuffix --> WriteConfig

    WriteConfig --> HasSnapshot{profile 有 auth snapshot?}
    HasSnapshot -->|否| SaveState[寫入 active profile state]
    HasSnapshot -->|是| BackupAuth[備份目前 auth.json]
    BackupAuth --> RestoreAuth[還原 profile 的 auth snapshot]
    RestoreAuth --> SaveState

    SaveState --> RestartEnabled{允許重啟 Codex?}
    RestartEnabled -->|否 測試模式| Done([完成])
    RestartEnabled -->|是| Terminate[終止目前 Codex process]
    Terminate --> OpenCodex[重新開啟 Codex.app]
    OpenCodex --> Done

    WriteConfig --> PrivatePerm[套用 0600 權限]
    RestoreAuth --> PrivatePerm

    classDef startEnd fill:#E6E6FA,stroke:#333,stroke-width:2px,color:#1B1B4D
    classDef process fill:#D3F9D8,stroke:#2B8A3E,stroke-width:2px,color:#0B3D18
    classDef decision fill:#FFF3BF,stroke:#A15C00,stroke-width:2px,color:#1F1F1F
    classDef security fill:#FFE3E3,stroke:#C92A2A,stroke-width:2px,color:#3B0A0A

    class Start,Done startEnd
    class BackupConfig,AddSuffix,WriteConfig,BackupAuth,RestoreAuth,SaveState,Terminate,OpenCodex process
    class BackupName,HasSnapshot,RestartEnabled decision
    class PrivatePerm security
```

## Profile 編輯與重新命名流程

```mermaid
flowchart TD
    Start([使用者按下 Save]) --> TrimName[清理前後空白]
    TrimName --> EmptyName{name 是否為空?}
    EmptyName -->|是| Stop([UI 禁用儲存])
    EmptyName -->|否| UniqueName[產生唯一顯示名稱]
    UniqueName --> SafeFile[用 safeFileName 產生安全檔名]
    SafeFile --> FileCollision{安全檔名是否碰撞?}
    FileCollision -->|是| AddIndex[顯示名稱加尾碼後重試]
    AddIndex --> SafeFile
    FileCollision -->|否| WriteNew[寫入新 TOML 檔]
    WriteNew --> AuthMove{檔名改變且有 auth snapshot?}
    AuthMove -->|是| MoveAuth[移動 auth snapshot 到新檔名]
    AuthMove -->|否| RemoveOld[刪除舊 TOML 檔]
    MoveAuth --> RemoveOld
    RemoveOld --> Reload[重新載入 profile 清單]
    Reload --> ReturnProfile[回傳 reload 後的 profile]
    ReturnProfile --> Done([完成並更新 UI 選取])

    classDef startEnd fill:#E6E6FA,stroke:#333,stroke-width:2px,color:#1B1B4D
    classDef process fill:#D0EBFF,stroke:#1C7ED6,stroke-width:2px,color:#102A43
    classDef decision fill:#FFF3BF,stroke:#A15C00,stroke-width:2px,color:#1F1F1F

    class Start,Stop,Done startEnd
    class TrimName,UniqueName,SafeFile,AddIndex,WriteNew,MoveAuth,RemoveOld,Reload,ReturnProfile process
    class EmptyName,FileCollision,AuthMove decision
```

## 登入快照捕捉與還原 Sequence

```mermaid
sequenceDiagram
    actor User as 使用者
    participant UI as ProfileEditorView
    participant Store as ProfileStore
    participant Auth as ~/.codex/auth.json
    participant Snap as profile-switcher/auth/*.auth.json
    participant Config as ~/.codex/config.toml
    participant Codex as Codex.app

    User->>UI: Capture Current Login
    UI->>Store: captureCurrentAuth(profile)
    Store->>Auth: 檢查 auth.json 是否存在
    alt auth.json 存在
        Store->>Auth: 讀取登入資料
        Store->>Snap: 寫入 profile auth snapshot 並套用 0600
        Store-->>UI: 顯示 Captured current Codex login
    else auth.json 不存在
        Store-->>UI: 顯示需要 file-based auth 的提示
    end

    User->>UI: Apply & Restart
    UI->>Store: update(profile, name, contents)
    Store-->>UI: 回傳 reload 後 profile
    UI->>Store: apply(profile)
    Store->>Config: 備份並寫入 profile TOML
    opt profile 有 auth snapshot
        Store->>Auth: 備份目前 auth.json
        Store->>Snap: 讀取 profile auth snapshot
        Store->>Auth: 寫回 auth.json 並套用 0600
    end
    Store->>Codex: terminate / openApplication
    Store-->>UI: 更新 active profile 與狀態訊息
```

## Release 與 Homebrew cask 更新流程

```mermaid
flowchart TD
    Version([決定 release version]) --> UpdateVersion[更新 Xcode MARKETING_VERSION<br/>與 cask version]
    UpdateVersion --> Test[本機執行 make test-unsigned]
    Test --> Build[本機執行 make dmg]
    Build --> Sign[Developer ID 簽署 app 與 DMG]
    Sign --> Notarize[提交 Apple notarization 並 staple]
    Notarize --> Gatekeeper[執行 stapler validate 與 spctl]
    Gatekeeper --> Checksum[計算最終 DMG SHA-256]
    Checksum --> UpdateCask[更新 Homebrew cask sha256]
    UpdateCask --> Commit[git commit -F 提交 release 變更]
    Commit --> Push[推送 main 與 v* tag 到 fork]
    Push --> Release[gh release create<br/>上傳 DMG 與 sha256]
    Release --> BrewFetch[brew fetch 驗證 cask URL 與 checksum]
    BrewFetch --> Done([發行完成])

    classDef startEnd fill:#E6E6FA,stroke:#333,stroke-width:2px,color:#1B1B4D
    classDef process fill:#D3F9D8,stroke:#2B8A3E,stroke-width:2px,color:#0B3D18

    class Version,Done startEnd
    class UpdateVersion,Test,Build,Sign,Notarize,Gatekeeper,Checksum,UpdateCask,Commit,Push,Release,BrewFetch process
```

## 閱讀建議

- 想理解程式碼責任分界：先看「整體架構」。
- 想修改套用 profile 行為：先看「Profile 套用流程」與 `ProfileStore.apply(_:)`。
- 想修改重新命名或檔名規則：先看「Profile 編輯與重新命名流程」。
- 想維護登入切換：先看「登入快照捕捉與還原 Sequence」。
- 想修改發行流程：先看「Release 與 Homebrew cask 更新流程」。
