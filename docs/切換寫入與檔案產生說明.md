# 切換寫入與檔案產生說明

## 切換時會寫入哪些參數

切換 profile 時，Codex Profile Switcher 不會逐項產生、合併或改寫特定參數。程式會把選定 profile 的整份 TOML 內容原樣寫入：

```text
~/.codex/config.toml
```

核心流程位於 `ProfileStore.apply(_:)`：

```swift
try ensureBackup()
try write(profile.contents, to: configURL)
try applyAuthSnapshotIfAvailable(for: profile)
```

其中 `profile.contents` 就是 profile TOML 檔的完整文字內容。因此，實際寫入 `config.toml` 的參數完全取決於該 profile 目前保存的 TOML。

例如，若 profile 內容是：

```toml
model = "gpt-5.5"
model_reasoning_effort = "high"
cli_auth_credentials_store = "file"
```

切換後 `~/.codex/config.toml` 就會變成這整份內容。

若 profile 內容包含 Azure OpenAI、Ollama 或其他 provider 設定，這些設定也會整份寫入。程式本身不會自動補齊或過濾任何 key。

## 是否會產生 `[profile].config.toml`

不會。

目前程式不會產生這種檔名格式：

```text
[profile].config.toml
```

實際產生的是一般 profile TOML 檔，儲存在：

```text
~/.codex/profile-switcher/profiles/
```

檔名格式是：

```text
<安全化後的 profile 名稱>.toml
```

範例：

```text
~/.codex/profile-switcher/profiles/OpenAI-Account.toml
~/.codex/profile-switcher/profiles/Azure-OpenAI-API.toml
~/.codex/profile-switcher/profiles/Ollama-GLM-5.2-Cloud.toml
```

檔名由 `safeFileName(for:)` 產生。它會保留英數字、`-`、`_` 與空白，其他字元會替換成 `-`，最後把空白轉成 `-` 並加上 `.toml`。

## 切換或設定過程可能產生的檔案

下表列出目前程式可能建立或更新的檔案：

| 類型 | 路徑或格式 | 產生時機 |
|---|---|---|
| 實際 Codex 設定 | `~/.codex/config.toml` | 套用 profile 時整份覆寫 |
| Profile TOML | `~/.codex/profile-switcher/profiles/<profile>.toml` | 建立、複製、重新命名或編輯 profile 時 |
| Config 備份 | `~/.codex/config.toml.profile-switcher-backup-yyyyMMdd-HHmmss` | 套用 profile 前，若 `config.toml` 存在 |
| Auth 檔案 | `~/.codex/auth.json` | Codex 使用檔案式 credential storage 時產生；套用含登入快照的 profile 時可能被覆寫 |
| Auth 備份 | `~/.codex/auth.json.profile-switcher-backup-yyyyMMdd-HHmmss` | 套用含登入快照的 profile 前，若 `auth.json` 存在 |
| Auth snapshot | `~/.codex/profile-switcher/auth/<profile-base-name>.auth.json` | 使用 `Capture Current Login` 捕捉目前登入時 |
| Active state | `~/.codex/profile-switcher/state.json` | 套用 profile 或更新 active profile 狀態時 |

若同一秒內已存在相同備份檔名，程式會自動加上遞增尾碼，例如：

```text
~/.codex/config.toml.profile-switcher-backup-20260622-101530-2
~/.codex/config.toml.profile-switcher-backup-20260622-101530-3
```

## 權限行為

以下檔案寫入後會套用私人檔案權限 `0600`：

- Profile TOML。
- `~/.codex/config.toml`。
- `auth.json` snapshot。
- `config.toml` 備份檔。
- `auth.json` 備份檔。

這個設計是為了避免其他本機使用者讀取可能含 provider endpoint、帳號資訊或 access token 的內容。

## 總結

- 切換 profile 時，寫入 `config.toml` 的是整份 profile TOML 內容。
- 程式不會依 key 合併、追加或局部更新 `config.toml`。
- 程式不會產生 `[profile].config.toml`。
- 多個 profile 會以多個 `<profile>.toml` 儲存在 `~/.codex/profile-switcher/profiles/`。
- 登入快照另存在 `~/.codex/profile-switcher/auth/`，不會寫進 profile TOML。
